1. Resumen Ejecutivo

1.1 Objetivo del Proyecto

Sistema web de monitoreo y control en tiempo real de cámaras frigoríficas de maduración de bananas, desarrollado para Almar SRL. El sistema centraliza la información de múltiples sensores y permite la visualización y control remoto desde dispositivos móviles y de escritorio.

1.2 Alcance

  • 30 cámaras frigoríficas monitoreadas
  • 60 sensores NUONUO (CO₂ y Etileno)
  • Sistema de autenticación con roles diferenciados
  • Dashboard responsive con actualización automática
  • Control de setpoints para operarios autorizados

1.3 Tecnologías Principales

  • Backend: ASP.NET Core 8.0 MVC
  • Base de Datos: SQLite con Entity Framework Core
  • Frontend: Razor Views, Tailwind CSS, jQuery
  • APIs Externas: Sitrad PRO API, NUONUO API

2. Descripción del Proyecto

2.1 Contexto

El proyecto surge de la necesidad de centralizar el monitoreo de variables críticas en cámaras de maduración de bananas, que anteriormente se consultaban de forma manual y distribuida en interfaces locales (HMI).

2.2 Problema Resuelto

  • Antes: Consultas manuales en interfaces locales, coordinación distribuida, falta de histórico
  • Después: Dashboard centralizado, acceso remoto, actualización en tiempo real, registro histórico (planificado)

2.3 Variables Monitoreadas

Por cada cámara se monitorean:

  • Temperatura de aire (Sitrad)
  • Temperatura de pulpa (Sitrad)
  • Humedad relativa (Sitrad)
  • Estado del evaporador (Sitrad)
  • CO₂ (NUONUO)
  • Etileno (NUONUO)
  • Setpoints SET1 y SET3 (Sitrad)

3. Arquitectura del Sistema

3.1 Arquitectura de Tres Capas

El sistema se estructura en tres capas principales:

Capa 1: Sensado y Control (Hardware)

  • Cámaras frigoríficas (30 unidades)
  • Sensores Sitrad (temperatura, humedad, evaporador)
  • Controladores Sitrad (concentradores de datos)
  • Sensores NUONUO (60 sensores: CO₂ y Etileno)

Capa 2: Backend (ASP.NET Core MVC)

Controladores:

  • SitradController: Dashboard principal, obtención de datos, actualización de setpoints
  • AccountController: Autenticación, registro, gestión de sesiones
  • HomeController: Página de inicio

Servicios:

  • AuthService: Gestión de usuarios, autenticación, hash de contraseñas
  • SitradApiClient: Cliente HTTP para API de Sitrad PRO
  • NuonoApiClient: Cliente HTTP para API de NUONUO

Modelos:

  • User: Modelo de usuario con roles
  • CameraViewModel: Modelo de vista para datos de cámaras
  • UserRole: Enum (Operario, Visualizador)

Persistencia:

  • AppDbContext: Contexto de Entity Framework Core
  • SQLite (sitrad.db): Base de datos local

Capa 3: Presentación (Dashboard Web)

Vistas Razor:

  • Login.cshtml: Formulario de autenticación
  • Register.cshtml: Formulario de registro
  • Index.cshtml: Dashboard principal con tabla de cámaras
  • _Layout.cshtml: Layout compartido

Frontend:

  • jQuery 3.6.0: Manipulación DOM y AJAX
  • Tailwind CSS: Estilos responsive
  • Iconify: Iconos SVG
  • Actualización automática cada 5 segundos

3.2 Flujo de Datos

Sensores → Controladores → APIs Externas → Backend → Frontend → Usuario
  1. Sensores generan lecturas físicas
  2. Controladores Sitrad/NUONUO envían datos a APIs
  3. Backend consume APIs mediante HttpClient
  4. Servicios combinan y procesan datos
  5. Controladores renderizan vistas
  6. Frontend actualiza dashboard mediante AJAX

3.3 Flujo de Autenticación

Usuario → Login → AccountController → AuthService → SQLite → Sesión → Dashboard
  1. Usuario accede y es redirigido a login
  2. AccountController recibe credenciales
  3. AuthService valida contra base de datos
  4. Si es válido, se crea sesión con UserId y Role
  5. Redirección al dashboard con permisos según rol
  6. Validación de permisos en cada acción

5. Funcionalidades Implementadas

5.1 Autenticación y Autorización

✅ Sistema de login/registro

  • Formulario de login con validación
  • Registro de nuevos usuarios
  • Hash SHA256 de contraseñas
  • Sesiones con timeout de 20 minutos

✅ Roles de usuario

  • Operario: Puede modificar setpoints y parámetros
  • Visualizador: Solo lectura, sin permisos de modificación

5.2 Dashboard

✅ Visualización en tiempo real

  • Tabla de cámaras con todas las variables
  • Actualización automática cada 5 segundos
  • Indicadores visuales de estado

5.3 Control de Setpoints

✅ Modificación de SET1 y SET3

  • Solo para usuarios con rol Operario
  • Validación de formato numérico
  • Sistema de reintentos automáticos
  • Confirmación visual mediante toasts

5.6 Funcionalidades Planificadas

  • ⏳ Historial de lecturas (logs)
  • ⏳ Agregaciones estadísticas (mínimo, máximo, promedio)
  • ⏳ Sistema de alertas avanzado con notificaciones
  • ⏳ Reportes exportables (PDF, Excel)
  • ⏳ Buffer local para cortes de red

7. Modelo de Datos

7.1 Tabla Users

Campo Tipo Descripción Restricciones
Id INTEGER Identificador único PK, AutoIncrement
Username TEXT Nombre de usuario NOT NULL, UNIQUE, Case-insensitive
PasswordHash TEXT Hash SHA256 de la contraseña NOT NULL
Role TEXT Rol del usuario NOT NULL, Enum: "Operario" o "Visualizador"
CreatedAt TEXT Fecha de creación NOT NULL, Default: datetime('now')

8. Configuración y Despliegue

8.1 Requisitos Previos

  • .NET 8.0 SDK o superior
  • Navegador web moderno (Chrome, Firefox, Edge, Safari)
  • Acceso a la red donde está la API de Sitrad PRO
  • NGROK (opcional, para exposición a Internet)

8.2 Configuración Inicial

Paso 1: Clonar repositorio

git clone https://github.com/GonzaloBelvisi/CloudPepeNew.git
cd CloudPepeNew-main

Paso 2: Configurar appsettings.json

cp appsettings.example.json appsettings.json

9. Seguridad

9.1 Autenticación

  • Hash de contraseñas: SHA256 (no se almacenan en texto plano)
  • Sesiones: Cookies HTTP-only y seguras
  • Timeout: 20 minutos de inactividad

9.2 Autorización

  • Control de acceso basado en roles: Validación en servidor y cliente
  • Código de invitación: Previene registro no autorizado de operarios
  • Validación de permisos: En cada acción que requiere privilegios

11. Guías de Uso

11.1 Primer Acceso

  1. Abrir navegador en http://localhost:5220
  2. Serás redirigido automáticamente a /Account/Login
  3. Usuario inicial: admin
  4. Contraseña: La configurada en appsettings.json bajo AuthSettings:AdminPassword
